Strategic Hybrid Cloud Transition Plan for SynesthorTech

A five-step transition plan for moving selected analytics workloads from legacy infrastructure into a secure and scalable hybrid cloud.

← Back to portfolio View résumé

Project objective

Address slow analytics, weak perimeter security, limited MFA, legacy databases, and cloud compatibility problems without disrupting necessary on-premises operations.

What I produced

  • Assessed hardware and software vulnerabilities in the existing environment.
  • Proposed AWS Systems Manager for unified operations, EMR for analytics, Aurora for database scaling, and EKS for containerized ETL workflows.
  • Recommended next-generation firewall and VPN upgrades with modern authentication.
  • Created a five-step transition plan covering requirements, assessment, architecture, phased migration, testing, optimization, and security.

Key decisions

  • Keep workloads on-premises when operational or compatibility needs require it.
  • Move compute-intensive analytics and scalable data processing to AWS.
  • Migrate in phases and validate performance, security, identity, and replication before proceeding.
  • Treat monitoring, patching, IAM, and configuration management as ongoing operations.

Validation and analysis

  • Mapped each modernization step to a documented problem in the current environment.
  • Defined why each workload belongs on-premises or in AWS.
  • Included testing and optimization checkpoints in the migration sequence.

What I would improve next

  • Add workload dependency mapping and a detailed data-migration runbook.
  • Define RTO, RPO, and rollback criteria for each phase.
  • Include cost estimates and a responsibility matrix for operations after migration.
