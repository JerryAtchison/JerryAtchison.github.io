CampusHub Mobile and Ubiquitous Design

Project objective

Reduce fragmentation across university systems by giving students one mobile platform for academic, communication, navigation, resource, and campus-service needs.

What I produced

  • Defined problems, primary features, user growth, data handling, performance, resource efficiency, and backend architecture.
  • Created personas representing different student needs and technical comfort levels.
  • Evaluated mobile-device capabilities, sensors, communication protocols, privacy, security, and accessibility.
  • Built a 30-page interactive PDF prototype with clickable navigation for dashboard, courses, messages, events, maps, resources, account features, security flows, and accessibility settings.

Key decisions

  • Keep heavy processing and sensitive records in a secure cloud backend rather than on the device.
  • Limit GPS and background synchronization to preserve battery and privacy.
  • Use secure APIs to integrate academic records and campus systems.
  • Design the first release as an MVP that can expand into payments, reservations, event registration, and IoT resource tracking.

Validation and analysis

  • Mapped features and clickable navigation paths to user personas and campus problems.
  • Reviewed performance during registration, exams, and high-traffic events.
  • Evaluated privacy, device limitations, communication methods, and iterative usability testing.

What I would improve next

  • Conduct usability tests with actual students and accessibility users.
  • Apply usability findings to the existing clickable prototype and formalize a reusable visual design system.
  • Define API contracts, authentication flows, and data-retention policies.
